# 项目优化计划书

## 一、项目现状分析

### 1. 存在问题

1. **问答模式API调用失败**：老师只能回答"抱歉，老师现在有点累，我们等会儿再继续好不好？"，疑似API配置错误
2. **教学模式API调用失败**：同样出现上述错误，API未调对
3. **备课计划功能无反应**：点击生成教案没有反应
4. **模块冗余**：项目计划和技能管理模块不需要

### 2. 问题原因分析

1. **API调用失败原因**：
   - `dialogue_manager.py`中的`_generate_response`方法在vLLM调用失败时返回默认错误消息
   - `inference.py`中默认使用local模式的VLLMClient，连接localhost:8000，但该服务可能未运行或配置错误
   - 未配置正确的API服务（如NVIDIA或Silicon Flow）

2. **备课计划无反应原因**：
   - `lesson_service.py`中同样存在API调用失败问题
   - 前端可能未正确处理API返回的错误情况

3. **模块冗余**：
   - `project.py`（项目计划模块）和`skill.py`（技能管理模块）不需要

## 二、优化方案

### 1. 修复API调用问题

#### 1.1 配置API服务
- **方案A**：使用NVIDIA API（推荐）
  - 优点：性能稳定，适合教育场景
  - 配置步骤：
    1. 在`.env`文件中设置`API_TYPE=nvidia`
    2. 确保`NVIDIA_API_KEY`已正确配置

- **方案B**：使用Silicon Flow API
  - 优点：国内访问速度快
  - 配置步骤：
    1. 在`.env`文件中设置`API_TYPE=silicon_flow`
    2. 确保`SILICON_FLOW_API_KEY`已正确配置

- **方案C**：本地部署vLLM服务
  - 优点：完全控制，无API调用费用
  - 缺点：需要足够的硬件资源
  - 配置步骤：
    1. 运行`model_service/start_all.py`启动本地服务
    2. 确保服务在localhost:8000正常运行

#### 1.2 代码优化
- 修改`dialogue_manager.py`中的错误处理逻辑，提供更友好的错误提示
- 修改`lesson_service.py`中的错误处理，确保即使API调用失败也能返回合理的默认教案

### 2. 移除冗余模块

- 移除`app/api/project.py`（项目计划模块）
- 移除`app/api/skill.py`（技能管理模块）
- 更新`app/api/__init__.py`，移除相关蓝图注册
- 更新前端代码，移除对这些模块的引用：
  - 移除`app/static/js/app.js`中的`loadSkills`、`useSkill`、`refreshSkills`、`generateProjectPlan`方法
  - 移除`updateSidebar`方法中的`project`和`skills`模式处理
  - 移除`app/templates/index.html`中导航栏的"项目计划"和"技能管理"按钮
  - 移除相关的HTML元素和事件绑定

### 3. 前端优化

- 优化备课计划页面的错误处理，确保即使API调用失败也能显示友好的提示
- 改进用户界面，提供更好的用户体验
