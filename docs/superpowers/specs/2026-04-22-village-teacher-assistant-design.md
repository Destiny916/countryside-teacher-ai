## 一、项目定位

### 1.1 系统概述

这是一个面向**小学阶段（6-12岁）**的AI助教系统，采用混合模式设计，支持：

1. **线上实时语音教学**：AI虚拟老师通过语音实时授课，学生可随时打断提问
2. **离线学习辅助**：学生自学时遇到问题，AI老师即时解答并详细讲解
3. **教师备课工具**：自动生成符合标准的教案、配套练习题、项目计划书
4. **题库问答**：拍照识别题目，自动给出答案和分步讲解

### 1.2 核心用户

- **小学生**：主要交互者，需要语音驱动的友好界面
- **乡村教师**：备课辅助，降低工作量
- **学校管理者**：项目计划书生成，数据看板

### 1.3 技术路线

- **AI能力**：基于微调开源模型（Qwen2.5-7B + LoRA微调）
- **部署环境**：网络稳定但带宽有限，RTX 3060 GPU（12GB显存）
- **核心原则**：本地推理为主，降低带宽依赖

## 三、核心功能详解

### 3.1 实时语音教学

#### 3.1.1 交互流程

```
学生语音输入 → ASR识别 → 对话管理
                              ↓
                    检测打断关键词（"停"等）
                           ↓
              ┌─────────打断处理─────────┐
              ↓                         ↓
          暂停授课                   理解问题
              ↓                         ↓
          记录断点                   生成解答
              ↓                         ↓
          TTS回复                   询问继续
              ↓                         ↓
        从断点恢复                   继续回答
```

#### 3.1.2 打断处理机制

- **打断关键词**：支持"停"、"等一下"、"老师停"、"不懂"、"再说一遍"等
- **断点记忆**：记录当前讲解进度（知识点位置、讲解深度）
- **恢复机制**：回复完学生问题后，自动询问"可以继续吗？"确认后从断点继续
- **上下文关联**：打断后的问题能关联到之前讲解的内容

#### 3.1.3 语音配置

| 参数 | 配置 | 说明 |
|------|------|------|
| 语速 | 0.8x-1.2x可调 | 默认0.9x，适合儿童 |
| 音色 | 女童声/男童声可选 | 亲切友好 |
| 语调 | 活泼型/温柔型 | 根据学科切换 |

### 3.2 智能备课系统

#### 3.2.1 数据训练体系

**训练数据来源**：
- 公开教育资源（人教版、苏教版教材）
- 历年优秀教案（版权授权）
- 教育部官方教案模板
- 历年考试真题和课后习题
- 优秀教师课堂实录

**三大训练任务**：

| 任务 | 输入 | 输出 | 微调目标 |
|------|------|------|---------|
| 教案生成 | 课程标准+教材章节 | 完整教案文档 | 结构规范、知识点分布合理 |
| 习题生成 | 教案内容 | 选择/填空/解答题+答案 | 难度分级、知识点覆盖 |
| 讲解生成 | 知识点 | 分步讲解+比喻例子 | 童趣化、步骤清晰 |

#### 3.2.2 RAG知识库

- 向量化存储：Chroma向量数据库
- 知识切片：按知识点章节切割
- 检索策略：混合检索（语义+关键词）

### 3.3 题库问答

#### 3.3.1 工作流程

```
拍照/扫描 → OCR文字识别 → 题目结构化
                                  ↓
                        调用微调模型推理
                                  ↓
                        生成"答案+分步讲解"
                                  ↓
                        学生追问 → 进入对话答疑
```

#### 3.3.2 支持题型

- 选择题：选项分析、排除法讲解
- 填空题：知识点关联、记忆技巧
- 解答题：步骤分解、格式规范

### 3.4 项目计划书生成

#### 3.4.1 触发方式

- 学校基本信息输入（学生人数、教师数量等）
- 需求描述（建设目标、预算范围等）
- 模板选择（基础设施建设/信息化升级/混合方案）

#### 3.4.2 输出内容

- 建设目标与背景
- 技术方案设计
- 设备清单与预算
- 实施进度表
- 风险评估与对策
- 预期效果与评估指标

## 四、技术选型

### 4.1 模型配置（RTX 3060适配）

| 能力 | 模型 | 配置 | 显存占用 |
|------|------|------|---------|
| LLM推理 | Qwen2.5-7B | INT4量化(AWQ) | ~8GB |
| 语音识别 | SenseVoice-small | 全精度 | ~2GB |
| 语音合成 | CosyVoice-small | 全精度 | ~1.5GB |
| 向量检索 | Embedding模型 | 全精度 | ~1GB |
| **总占用** | | | **~12.5GB** |

### 4.2 框架与工具

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 后端框架 | Flask | 轻量易扩展 |
| 模型推理 | vLLM | 高吞吐量、流式输出 |
| 向量数据库 | Chroma | 轻量本地部署 |
| OCR识别 | PaddleOCR | 中文支持好 |
| RAG框架 | LangChain | 模块化 |
| 数据库 | SQLite | 零运维 |

### 4.3 显存优化策略

1. **INT4量化**：AWQ量化，质量损失<3%
2. **pagedattention**：vLLM显存分页管理
3. **动态batch**：按需分配显存
4. **CPU offload**：部分层卸载到内存（必要时）

## 六、数据存储

### 6.1 存储结构

```
/data
├── user_data.db          # 用户数据、会话记录
├── knowledge_base/        # 知识库向量数据
│   ├── embeddings/       # 向量文件
│   └── metadata/         # 元数据
├── lesson_materials/     # 教案、课件文件
├── training_data/        # 训练数据集
│   ├── raw/             # 原始数据
│   └── processed/       # 处理后数据
└── models/               # 模型权重
    ├── base/            # 基础模型
    └── finetuned/       # 微调后模型
```

### 6.2 数据库表结构

| 表名 | 内容 |
|------|------|
| users | 用户信息（学生/老师/管理员） |
| conversations | 对话记录 |
| learning_progress | 学习进度 |
| lesson_plans | 生成的教案 |
| questions | 题库 |
